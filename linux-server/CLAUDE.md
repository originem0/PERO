# Linux 服务器基础

## 学习信息
- **目标**：在 Linux 服务器上能独立干活（部署、管理、排查问题）
- **策略**：Windows 对比法——每个概念先找 Windows 对应物，再看差异和原因

## 逻辑链

### 模块 1：Shell 与命令行基础
1. 操作系统 = Kernel（管硬件）+ Shell（人机交互层）
2. Shell 本身是一个程序，权限取决于当前用户
3. 所有程序（包括 Shell）都通过 Kernel 操作硬件
4. Shell 的工作流：接收文本 → 解析 → 通过 PATH 找程序 → 启动程序 → 返回结果
5. 命令结构：`命令名 选项 参数`，Shell 按空格拆分全部传给程序，程序自己解析
6. 内置命令（如 cd）vs 外部程序（如 ls）：需要改变 Shell 自身状态的必须内置
7. 管道 `|` 串联小程序，重定向 `>` 输出到文件 → Linux 哲学：小工具组合

### 模块 2：文件系统与目录结构
1. Linux 只有一个根目录 `/`，所有内容从此往下，像一棵倒着的树
2. 没有盘符 → 存储设备通过挂载接入目录树某个位置 → 路径与物理设备解耦（换硬盘不影响程序）
3. "一切皆文件"：普通文件、目录、硬件设备都被抽象为文件 → 统一用读写操作
4. 硬件变文件的机制：Kernel 的驱动程序负责把文件读写翻译成具体硬件操作
5. 目录按类型组织（程序→`/usr/bin`、配置→`/etc`、日志→`/var/log`、设备→`/dev`、用户→`/home`）→ 服务于管理员视角，形成规范，任何人都能预判文件位置
6. 路径两种写法：绝对路径从 `/` 开始，相对路径从当前目录开始；`.` 当前目录，`..` 上一级

### 模块 3：用户与权限
1. 服务器需要多用户 → 安全隔离（人与人之间、服务与服务之间）
2. 最小权限原则：每个用户/服务只给刚好够用的权限，**不只是防人，也是防程序**——每个服务用自己的用户运行，一个被攻破不连累其他
3. root 是超级管理员不受权限限制，普通用户只能干被允许的事
4. 每个文件标记：owner（一个用户）+ group（一个组）+ 三组 rwx 权限
5. 系统按身份匹配：你是 owner？→ 第一组；在 group 里？→ 第二组；都不是？→ 第三组（others）
6. 权限绑在文件上，不是绑在用户上——同一个用户对不同文件可能是不同角色
7. chmod 改文件本身的权限设置，只有 owner 和 root 能改

### 模块 4：进程管理
1. 程序是硬盘上的静态文件，进程是程序加载到内存运行的实例
2. 同一个程序可以启动多个进程（如多个 Chrome 窗口）
3. 每个进程有独立的内存空间、工作目录、环境变量，Kernel 强制隔离
4. 进程隔离的原因：稳定性（一个崩了不影响其他）+ 安全性（看不到彼此数据）
5. cd 必须内置的根因：进程隔离 → 独立程序改不了 Shell 进程的工作目录
6. Shell 是长期运行的进程，它启动的 ls、grep 等是短命进程，干完就退出
7. ps 查快照、top 实时监控、kill PID 杀进程；PID 1 是系统第一个进程（systemd）

### 模块 5：包管理与软件安装
1. 包管理器（apt）做的事：从远程中央仓库下载 + 自动处理依赖 + 自动把文件放到正确位置（/usr/bin、/etc、/var/log）
2. 三种安装方式：包管理器（全自动）> .deb 安装包（半自动，不自动下载依赖）> 压缩包解压（全手动）
3. Linux 没有注册表，配置就是 /etc 下的文本文件——"一切皆文件"的延伸
4. 仓库地址记录在系统配置里，可以添加第三方仓库

### 模块 6：网络基础
1. IP 找机器，端口找程序；任何接收网络请求的程序都需要监听 IP + 端口
2. 一台机器可以有多个网络接口，每个有自己的 IP，通往不同网络
3. `127.0.0.1` 只限本机、`0.0.0.0` 监听所有接口——程序选择监听哪个接口决定谁能访问它
4. 一个端口同一时间只能被一个程序占用
5. 反向代理（nginx）挡在前面：统一接待、安全隔离、负载均衡；后端服务只监听 127.0.0.1
6. 排查链从内到外：程序在跑吗(ps) → 端口在监听吗(ss) → 本机能访问吗(curl localhost) → 网络通吗(ping) → 防火墙挡了吗
7. curl = 命令行浏览器，用于测试 HTTP 服务和 API

### 模块 7：服务管理（systemd）
1. 服务 = 在后台持续运行的进程（nginx、sshd 等），区别于 ls 这种跑完就退出的短命进程
2. systemd 是 PID 1，系统的服务管家，负责按配置启动、停止、监控所有服务
3. 程序要被 systemd 管理，必须有 .service 配置文件注册（apt 安装的服务类软件会自动创建）
4. systemctl 是跟 systemd 交互的命令：start/stop/restart/status/enable/disable
5. start（立刻启动）和 enable（开机自启动）是两件独立的事，新服务通常两个都要执行
6. systemctl status 可以看到：运行状态、运行时长、PID、是否开机自启动

### 模块 8：基本安全实践
1. 安全的核心原则 = 最小权限：能不暴露的就不暴露
2. 默认的 root + 密码 + 22 端口是最不安全的组合——攻击者只需猜密码
3. SSH 加固三件套：密钥登录禁用密码 + 禁止 root 直接登录 + 改默认端口
4. 防火墙：只开放必要端口（80/443/SSH 端口），其余全封
5. 加固顺序原则：先确保新方式能用，再关旧方式——防止把自己锁在门外
