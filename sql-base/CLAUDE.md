## 学习信息
- **目标（解释模型）**：面对 Web 后端业务需求，能设计合理的表结构并说清为什么这样拆；面对慢查询，能读执行计划、定位瓶颈、判断怎么优化。
- **当前水平**：以前浅层学过 MySQL，现已遗忘，有残留概念框架可激活。
- **策略**：基础快速激活，重心放在 schema 设计和索引/执行计划两个核心判断力上。PostgreSQL 特性穿插教学，不单独分模块。
- **结构性洞察**：关系数据库的核心取舍——提前定死结构（schema）和表间关系（外键），换来整列统一操作能力和跨表关联可靠性。代价是 schema 变更成本高、不规整数据适配差。
