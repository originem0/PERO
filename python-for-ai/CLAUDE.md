## 逻辑链

### 模块 1：执行模型与开发环境
- Python 是解释型语言：源码 → 字节码(.pyc) → VM 逐条执行，始终需要解释器在场
- 编译型 vs 解释型本质区别：产物是 CPU 认识的机器码 vs 只有 VM 认识的字节码
- CPython 解释器 = 解析器 + VM，装 Python 就有了，VM 不是额外的东西
- 虚拟环境(venv)：项目依赖隔离工具，和执行流程无关，不创建就不存在
- JIT（如 JS V8）会把热代码升级为机器码，CPython 没有 JIT

### 模块 2：对象模型与数据类型
- 一切皆对象，全员引用——包括 int 在内，没有"原始类型"
- 对象三要素：identity（id()查看）、type（type()查看）、value
- 名字是标签不是盒子：标签不属于对象，一个对象可以有零到多个标签
- 可变(list/dict/set)：对象自身内容可被修改；不可变(int/float/bool/str/tuple/None)：创建后不能改
- b = a 是让 b 指向 a 指向的同一个对象，可变对象通过一个标签修改后另一个标签也能看到
- Python 通过引用计数自动管理对象释放

### 模块 3：名字、作用域与绑定
- LEGB查找顺序：Local(函数内) → Enclosing(外层函数) → Global(模块级) → Built-in(Python自带)
- 作用域在编译时（生成字节码时）确定，不是运行时逐行判断
- 函数内有赋值语句 → 该名字在整个函数内都是局部变量（不是从赋值行开始）
- 这就是为什么"先print后赋值"会报UnboundLocalError而不是去读全局
- global/nonlocal显式声明可以覆盖默认的局部行为
- 编译时确定作用域的原因：局部变量用数组存储(快)，全局变量用字典存储(慢)
